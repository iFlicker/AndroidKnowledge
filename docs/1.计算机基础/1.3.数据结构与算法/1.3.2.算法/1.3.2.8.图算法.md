# 1.3.2.8 图算法

图论（Graph Theory）是计算机科学与离散数学中最为核心的分支之一。在实际工程与学术研究中，图结构被广泛用于抽象社交网络、交通路网、通信拓扑、依赖关系等复杂关联系统。而图搜索与核心图算法，则是解决这些物理世界问题的重要数学工具。

本文将从数学机理与系统设计实现双重视角出发，深入剖析最小生成树（MST）、单源最短路径（SSSP）及多源最短路径（APSP）三类核心图算法。文章将详细推导各算法背后的数学定理、给出通用语言（如 C++）的高效实现、分析算法在不同图拓扑结构下的时空复杂度，并探讨其在现代系统工程中的应用与选型。

## 1. 图论基础与数学模型

在深入研究具体算法之前，我们需要明确图的数学定义与计算机内的表示方法。这些基础数据结构的优缺点直接决定了后续算法的时间与空间复杂度上限。

### 1.1 图的数学定义
图（Graph）由顶点（Vertex）集合 $V$ 和边（Edge）集合 $E$ 构成，记为 $G = (V, E)$。
- **有向图与无向图**：若边集 $E$ 中的元素是有序对 $\langle u, v \rangle$，则为有向图（Directed Graph），表示从 $u$ 到 $v$ 的单向连接；若为无序对 $(u, v)$，则为无向图（Undirected Graph）。
- **度（Degree）**：在无向图中，与顶点 $v$ 关联的边的条数称为该顶点的度，记为 $d(v)$。在有向图中，分为入度（In-degree, $id(v)$，指向该节点的边数）和出度（Out-degree, $od(v)$，从该节点指出的边数），满足 $d(v) = id(v) + od(v)$。
- **割（Cut）**：图 $G=(V, E)$ 的一个割 $(S, V-S)$ 是将顶点集 $V$ 划分为两个非空子集 $S$ 和 $V-S$ 的划分。
- **横跨边（Crossing Edge）**：若一条边的一端属于 $S$，另一端属于 $V-S$，则称这条边为横跨割 $(S, V-S)$ 的横跨边。

### 1.2 图的计算机存储结构剖析

不同的存储结构在处理稀疏图（Sparse Graph, $E \ll V^2$）和稠密图（Dense Graph, $E \approx V^2$）时具有完全不同的表现。

#### 1.2.1 邻接矩阵（Adjacency Matrix）
邻接矩阵使用一个二维数组 `g[V][V]` 来表示图。对于无权图，若存在边 $(i, j)$，则 `g[i][j] = 1`，否则为 `0`；对于带权图，若存在边 $(i, j)$，则 `g[i][j] = w`（边权），否则通常设为 $\infty$（无连接）。

*   **优点**：
    *   能在 $O(1)$ 时间内判断任意两点之间是否存在边。
    *   适合稠密图，存储紧凑，数组在内存中是连续分配的，具有极佳的 CPU 缓存局域性。
    *   对于无向图，邻接矩阵是对称矩阵，可以通过只存储上三角或下三角来将空间占用减半。
*   **缺点**：
    *   空间复杂度为 $O(V^2)$。当顶点数 $V$ 达到 $10^5$ 级别时，所需的内存大小（约 10 GB 整数空间）在常规单机系统上是无法承受的，即使此时边数 $E$ 非常少（稀疏图）。
    *   遍历某个顶点 $u$ 的所有邻接点需要 $O(V)$ 的时间，这会导致图遍历算法（如 DFS/BFS）的整体效率下降。

#### 1.2.2 邻接表（Adjacency List）
邻接表为每个顶点 $u$ 维护一个链表或动态数组，存放所有从 $u$ 出发的邻接点信息。

*   **优点**：
    *   空间复杂度为 $O(V + E)$，是表示稀疏图的首选方案。
    *   遍历一个节点的邻接点只需要 $O(d(u))$ 的时间，整体遍历全图的复杂度为 $O(V + E)$。
*   **缺点**：
    *   判断两点之间是否存在边需要遍历链表，最坏情况下需要 $O(V)$ 时间。
    *   在内存中通过指针连接各个节点，会导致频繁的非连续内存访问，引发大量的 CPU Cache Miss，在高性能计算场景下常数较大。

#### 1.2.3 链式前向星（Linked Forward Star）
为了解决传统链表邻接表 Cache Miss 高的问题，算法工程中常用“链式前向星”来存储图。它本质上是用静态数组模拟邻接表，将所有边存储在一个连续的数组中。

##### 结构设计与实现
```cpp
struct Edge {
    int to;     // 边的终点
    int next;   // 与当前边同起点的上一条边在 edge 数组中的下标
    int w;      // 边权
};

std::vector<Edge> edges;
std::vector<int> head; // head[u] 表示以 u 为起点的最后一条加入的边在 edges 数组中的下标

// 初始化
void init(int v_count) {
    head.assign(v_count, -1);
    edges.clear();
}

// 加边操作 (u -> v, 权重为 w)
void add_edge(int u, int v, int w) {
    Edge e = {v, head[u], w};
    edges.push_back(e);
    head[u] = edges.size() - 1; // 更新 head[u] 指向最新插入的边
}

// 遍历顶点 u 的所有出边
void traverse(int u) {
    for (int i = head[u]; i != -1; i = edges[i].next) {
        int v = edges[i].to;
        int weight = edges[i].w;
        // 处理边 (u, v)
    }
}
```
链式前向星不仅规避了频繁申请和释放内存节点的开销，而且通过静态数组存储，大幅提升了内存局部性，是实现高性能图算法的基石。

#### 1.2.4 十字链表（Orthogonal List）与邻接多重表（Adjacency Multilist）
*   **十字链表**：针对有向图。它将邻接表和逆邻接表结合起来，每个顶点维护两个链表头（出边链表、入边链表），每个边节点既包含起点和终点，也包含指向下一个同起点和同终点的指针。这使得同时查询一个节点的入边和出边变得极为高效。
*   **邻接多重表**：针对无向图。在邻接表中，无向图的一条边 $(u, v)$ 会被分别存放在 $u$ 和 $v$ 的邻接表中，导致同一条边被实例化两次，给“标记边是否被访问”等操作带来麻烦。邻接多重表将每条边抽象为一个节点，该节点同时被链入起点和终点的关联链表中，保证物理上每条边只存在一份。

### 1.3 切分定理（Cut Property）
切分定理是解决最小生成树（MST）问题的核心数学基石。理解并能够证明切分定理，是理解 Kruskal 和 Prim 等贪心算法合理性的前提。

> **切分定理（Cut Property）**：
> 设 $G = (V, E)$ 是一个带权无向连通图。令 $(S, V-S)$ 是 $G$ 的任意一个割。在所有横跨割 $(S, V-S)$ 的边中，如果边 $e = (u, v)$（其中 $u \in S, v \in V-S$）的权值 $w(e)$ 是唯一的最小值，那么图 $G$ 的任何最小生成树（MST）都必然包含这条边 $e$。

#### 数学证明（反证法）
假设存在图 $G$ 的一棵最小生成树 $T$，且 $T$ 不包含最小横跨边 $e = (u, v)$。

1.  由于 $T$ 是一棵生成树，它是连通的且无环。因此，在 $T$ 中必然存在唯一的一条连接 $u$ 和 $v$ 的简单路径 $P$。
2.  因为 $u \in S$ 且 $v \in V-S$，路径 $P$ 的起点在 $S$ 中，终点在 $V-S$ 中。根据介值原理，路径 $P$ 上必然至少存在一条边 $e' = (u', v')$，使得 $u' \in S$ 且 $v' \in V-S$。即 $e'$ 也是横跨割 $(S, V-S)$ 的一条边。
3.  我们将边 $e'$ 从树 $T$ 中删除。此时，树 $T$ 被分裂成两个互不相交的连通分量 $T_S$ 和 $T_{V-S}$。
4.  我们将最小横跨边 $e$ 加入。由于 $e$ 连接了 $T_S$ 中的顶点 $u$ 和 $T_{V-S}$ 中的顶点 $v$，我们重新构造出了一棵新的生成树 $T' = T - \{e'\} + \{e\}$。
5.  我们来对比这两棵树的权重和：
    $$W(T') = W(T) - w(e') + w(e)$$
6.  根据前提条件，$e$ 是横跨该割的所有边中权值唯一最小的边，而 $e'$ 也是横跨该割的边，因此必有：
    $$w(e) < w(e')$$
7.  从而推导出：
    $$W(T') < W(T)$$
    这与 $T$ 是图 $G$ 的最小生成树这一假设产生矛盾！
8.  如果最小横跨边不唯一，我们同样可以推导出 $W(T') \le W(T)$，即至少存在一棵包含该最小横跨边 $e$ 的最小生成树。

证明完毕。切分定理告诉我们：**任何一个割的最小横跨边，都是构建最小生成树的安全选择**。Kruskal 和 Prim 的贪心策略本质上都是在不断寻找不同割下的最小横跨边。

```
     割集划分面 (S | V-S)
       S          |        V-S
     ( u )--------|-------( v )   <-- 最小横跨边 e (被选入 MST)
      /           |           \
    /             |             \
  (u')------------|-----------(v')  <-- 另一条横跨边 e' (若选入会导致环并使总权值变大)
                  |
```

## 2. 最小生成树算法 (Minimum Spanning Tree - MST)

对于一个带权无向连通图，最小生成树（MST）是包含图中所有顶点且边权之和最小的极小连通子图（显然它是一棵树）。

### 2.1 Kruskal 算法（基于“加边”的贪心策略）

Kruskal 算法是一种典型的加边贪心算法。其核心思路是：将图中的所有边按权值从小到大排序，然后依次考虑每一条边。若当前边的两个顶点在已构建的森林中不属于同一个连通分量，则将该边加入生成树；否则丢弃。

#### 2.1.1 并查集（Disjoint Set / Union-Find）判环与合并优化
判断两个顶点是否属于同一个连通分量，本质上是集合的查询与合并问题。如果使用常规的图遍历算法（DFS/BFS）来判环，单次判断需要 $O(V)$ 的时间，这会使 Kruskal 的复杂度退化到 $O(E \cdot V)$。

为此，引入**并查集**这一数据结构，其支持两个核心操作：
- `find(x)`：查找元素 $x$ 所属集合的代表元素（根节点）。
- `unite(x, y)`：将元素 $x$ 和 $y$ 所在的集合合并。

为了确保并查集操作的高效性，必须引入以下两种优化：
1.  **路径压缩（Path Compression）**：在执行 `find` 寻找根节点的过程中，将路径上所有节点的父指针直接指向根节点。这使得后续的查询深度直接降低为常数级。
2.  **按秩合并（Union by Rank/Size）**：在合并两个集合时，总是将高度较小（秩较小）的树的根节点挂载到高度较大（秩较大）的树的根节点下。这防止了树在合并过程中退化为线性链表。

##### 优化并查集的复杂度分析
单独使用路径压缩或按秩合并，单次操作的最坏时间复杂度为 $O(\log V)$。
当**同时使用路径压缩与按秩合并**时，对于包含 $V$ 个元素的并查集，进行 $M$ 次操作的平摊时间复杂度仅为 $O(M \cdot \alpha(V))$。其中 $\alpha(V)$ 是**反阿克曼函数（Inverse Ackermann Function）**。
反阿克曼函数的增长速度慢到令人难以置信：对于现实世界中所有的合理输入 $V$，都有 $\alpha(V) \le 4$。因此，在工程实践中，并查集的单词操作可以视作常数时间 $O(1)$。

#### 2.1.2 Kruskal 算法流程与 C++ 实现

##### 算法步骤：
1.  将图 $G$ 中的所有边 $E$ 按权重从大到小（或从小到大）进行升序排序。
2.  初始化并查集，每个顶点各自独立为一个集合。
3.  遍历排好序的边集。对于每一条边 $(u, v)$，执行：
    *   调用 `find(u)` 和 `find(v)`，判断它们是否在同一集合。
    *   若不在（即不构成环），则将该边加入 MST 中，并调用 `unite(u, v)` 进行合并。
    *   若在（即构成环），则直接跳过该边。
4.  当加入的边数达到 $V-1$ 条时，算法可提前终止，MST 构建完成。

##### 完整代码实现
```cpp
#include <iostream>
#include <vector>
#include <algorithm>

struct Edge {
    int u, v, w;
    bool operator<(const Edge& other) const {
        return w < other.w;
    }
};

class DisjointSet {
private:
    std::vector<int> parent;
    std::vector<int> rank;

public:
    DisjointSet(int n) {
        parent.resize(n);
        rank.resize(n, 0);
        for (int i = 0; i < n; ++i) {
            parent[i] = i;
        }
    }

    int find(int i) {
        if (parent[i] == i)
            return i;
        return parent[i] = find(parent[i]); // 路径压缩
    }

    bool unite(int i, int j) {
        int root_i = find(i);
        int root_j = find(j);
        if (root_i != root_j) {
            // 按秩合并
            if (rank[root_i] < rank[root_j]) {
                parent[root_i] = root_j;
            } else if (rank[root_i] > rank[root_j]) {
                parent[root_j] = root_i;
            } else {
                parent[root_j] = root_i;
                rank[root_i]++;
            }
            return true;
        }
        return false;
    }
};

std::vector<Edge> kruskal(int v_count, std::vector<Edge>& edges) {
    std::vector<Edge> mst;
    std::sort(edges.begin(), edges.end()); // 按权值升序排序

    DisjointSet ds(v_count);
    int edges_used = 0;

    for (const auto& edge : edges) {
        if (ds.unite(edge.u, edge.v)) {
            mst.push_back(edge);
            edges_used++;
            if (edges_used == v_count - 1) {
                break;
            }
        }
    }
    return mst;
}
```

##### 复杂度分析
*   **时间复杂度**：
    *   边的排序：$O(E \log E)$。
    *   并查集判环与合并：共需遍历 $E$ 条边，每次进行两次 `find` 和至多一次 `unite`，耗时 $O(E \cdot \alpha(V))$。
    *   由于 $E \le V^2$，则 $\log E \le 2\log V$。因此，Kruskal 算法的整体时间复杂度受限于排序瓶颈，为 $O(E \log E)$，也可写作 $O(E \log V)$。
*   **空间复杂度**：需要存储边集以及并查集辅助数组，空间复杂度为 $O(V + E)$。

### 2.2 Prim 算法（基于“加点”的贪心策略）

与 Kruskal 算法关注边不同，Prim 算法是一个典型的“加点”贪心算法。它从图中的某一个起始顶点开始，逐步向外扩张，每次选择距离当前生成树集合最近的非树顶点，将其与连接该顶点的最小边加入树中，直到所有顶点均被纳入。

#### 2.2.1 优先队列（Min-Heap）优化与索引优先队列

在朴素的 Prim 算法中，每一次需要遍历所有非树节点以寻找距离当前生成树最近的节点，这需要 $O(V)$ 时间，总时间复杂度为 $O(V^2)$。

为了加速这一选择过程，我们引入**最小堆（Min-Heap / 优先队列）**。
- 堆中存放的数据为 `(key, vertex)` 对，其中 `key` 表示顶点 `vertex` 到当前已构建的 MST 集合的最小距离。
- 每次从堆顶取出（Pop）最小元素，可在 $O(\log V)$ 时间内找到最近的顶点。
- 找到顶点后，遍历其所有邻接点，若有更短的距离，则更新该邻接点在堆中的 `key` 值。

##### 懒惰删除（Lazy Deletion）与索引优先队列（Indexed Priority Queue）的选择
在标准的二叉堆中，更新堆中某个元素的权值并进行堆调整（称为 `decrease_key`）需要 $O(\log V)$ 的时间。然而，许多主流编程语言的标准库（如 C++ 的 `std::priority_queue`，Java 的 `PriorityQueue`）并没有直接暴露 `decrease_key` 接口。
在工程实现中，有两套成熟的方案：

1.  **懒惰删除（Lazy Deletion / 冗余入堆）**：
    每当发现一个顶点 $v$ 有更短的距离时，不修改堆中原有的记录，而是直接向优先队列中插入一条新的更优记录 `(new_dist, v)`。当该节点从队首弹出时，我们首先判断它是否已经被标记为“已访问”。如果是，则说明这是一条已经失效的冗余历史记录，直接抛弃即可。
    *   *特点*：实现极其简单，直接使用标准库内置的优先队列。但因为堆中存在重复顶点，堆的最大容量可达 $O(E)$，因此单次堆操作的复杂度为 $O(\log E)$。
2.  **索引优先队列（Indexed Priority Queue）**：
    在普通的二叉堆之上维护一个映射数组，记录每个顶点在堆中的具体下标位置。这样在需要修改顶点距离时，可以直接定位到对应的堆节点，并在 $O(\log V)$ 时间内完成上浮操作。
    *   *特点*：堆中的元素个数始终严格保持在 $O(V)$，理论常数更小，但需手动实现堆结构。
3.  **斐波那契堆（Fibonacci Heap）优化**：
    从理论而言，若使用斐波那契堆，`decrease_key` 的平摊复杂度为 $O(1)$，可以将 Prim 算法的整体复杂度降低至 $O(E + V \log V)$。然而，斐波那契堆内部结构非常复杂，其维护指针的常数极大，在实际的工业数据集规模下，其运行速度通常慢于优化后的二叉堆。

#### 2.2.2 堆优化 Prim 算法流程与 C++ 实现

##### 算法步骤（采用懒惰删除）：
1.  初始化：定义数组 `visited` 标记顶点是否已进入 MST，定义数组 `dist` 存放各顶点到当前 MST 的最小距离，并将所有 `dist` 设为 $\infty$。
2.  选择源点（如顶点 0），设 `dist[0] = 0`。将 `(0, 0)`（即 `(dist, vertex)`）压入最小堆中。
3.  当堆不为空且 MST 边数未达到 $V-1$ 时，执行：
    *   弹出堆顶元素 `(d, u)`。
    *   若 `visited[u]` 为真，说明此为冗余数据，直接跳过。
    *   否则，标记 `visited[u] = true`。此时边（对应前驱节点）正式并入 MST。
    *   遍历 $u$ 的所有邻接边 $(u, v, w)$：
        *   若 `visited[v]` 为假，且 $w < dist[v]$，则更新 `dist[v] = w`，并将新记录 `(dist[v], v)` 压入堆中。

##### 完整代码实现
```cpp
#include <iostream>
#include <vector>
#include <queue>

struct Edge {
    int to;
    int w;
};

// 堆节点定义
struct HeapNode {
    int dist;
    int u;
    bool operator>(const HeapNode& other) const {
        return dist > other.dist; // 最小堆
    }
};

std::vector<std::pair<int, int>> prim(int v_count, const std::vector<std::vector<Edge>>& adj) {
    std::vector<std::pair<int, int>> mst_edges; // 存放最终 MST 的边 (from, to)
    std::vector<int> dist(v_count, 1e9);        // 初始化距离为无穷大
    std::vector<int> parent(v_count, -1);       // 记录前驱节点以重构 MST
    std::vector<bool> visited(v_count, false);

    // 最小堆优先队列
    std::priority_queue<HeapNode, std::vector<HeapNode>, std::greater<HeapNode>> pq;

    dist[0] = 0;
    pq.push({0, 0});

    int count = 0; // 记录已确定的顶点数

    while (!pq.empty()) {
        HeapNode node = pq.top();
        pq.pop();

        int u = node.u;
        if (visited[u]) continue;

        visited[u] = true;
        count++;

        if (parent[u] != -1) {
            mst_edges.push_back({parent[u], u});
        }

        for (const auto& edge : adj[u]) {
            int v = edge.to;
            int w = edge.w;
            if (!visited[v] && w < dist[v]) {
                dist[v] = w;
                parent[v] = u;
                pq.push({dist[v], v});
            }
        }
    }

    if (count != v_count) {
        // 如果最终确定的顶点数少于 V，说明图不连通，无法生成完整的 MST
        return {};
    }

    return mst_edges;
}
```

##### 复杂度分析
*   **时间复杂度**：
    *   每个顶点都会被标记 `visited` 一次，其出边会被遍历一遍，共执行 $O(E)$ 次堆插入操作。
    *   采用懒惰删除时，堆中最多可能有 $E$ 个元素，每次插入与弹出的复杂度为 $O(\log E) = O(\log V)$。
    *   因此，总时间复杂度为 $O(E \log V)$。
*   **空间复杂度**：堆与辅助数组的空间开销为 $O(V + E)$。

### 2.3 Kruskal 与 Prim 的对比及适用场景

在工程落地中，选择哪种最小生成树算法需要评估图的拓扑结构、内存约束以及代码维护成本。

| 对比维度 | Kruskal 算法 | Prim 算法（堆优化） | Prim 算法（朴素版） |
| :--- | :--- | :--- | :--- |
| **算法核心思想** | 贪心加边（按边权排序） | 贪心加点（从一点向外辐射） | 贪心加点（从一点向外辐射） |
| **时间复杂度** | $O(E \log V)$ | $O(E \log V)$ | $O(V^2)$ |
| **空间复杂度** | $O(V + E)$ | $O(V + E)$ | $O(V)$ （邻接矩阵下） |
| **最适图类型** | 稀疏图 | 稠密图 | 极稠密图 |
| **非连通图处理** | 自动产生最小生成森林 | 只能求出起始节点所在的 MST | 只能求出起始节点所在的 MST |
| **数据结构依赖** | 并查集、数组排序 | 最小二叉堆/斐波那契堆 | 仅基本数组 |

#### 场景选型深究：
1.  **稀疏图（$E \approx V$）**：Kruskal 算法通常具有更佳的表现。其核心开销在于对边集的快速排序，对于小规模的边数，排序常数极小，且并查集底层实现是一段极其紧凑的连续内存，缓存命中率极高。
2.  **稠密图（$E \approx V^2$）**：
    *   当图极度稠密（如完全图）时，堆优化的 Prim 算法中堆的调整次数急剧增加，甚至退化。此时，**朴素版 $O(V^2)$ Prim** 的常数项极小，不需要维护复杂的堆结构，性能反而超越堆优化的 Prim 与 Kruskal。
    *   在稠密图下，Kruskal 排序的边数 $E$ 巨大，会导致 $O(E \log V)$ 变得非常缓慢。
3.  **多连通分量（Forest）**：如果图本身是不连通的，Kruskal 算法在遍历完所有边后会天然地得到一个“最小生成森林”。而 Prim 算法如果仅执行一次，只能得到一个连通分量的树，若要得到森林，必须在外层循环遍历顶点，对未访问节点重复启动算法，代码复杂度会相应增加。

## 3. 单源最短路径算法 (Single-Source Shortest Path)

单源最短路径（SSSP）问题是图论中最经典的算法问题：给定一个带权图 $G=(V, E)$ 和一个源点 $s$，求从 $s$ 到图中其他所有顶点的最短路径长度。

### 3.1 Dijkstra 算法

Dijkstra 算法是一种用于计算**无负权边**图上单源最短路径的贪心算法。其本质是利用已确定最短距离的顶点集合作为中继，通过松弛操作逐步向外确定其他顶点的最短距离。

#### 3.1.1 贪心选择策略的数学合理性证明
Dijkstra 的算法逻辑是：维护一个已确定最短路径的顶点集合 $S$。在每一步中，选择未确定集合 $V-S$ 中距离源点 $s$ 最近的顶点 $u$，将其并入 $S$。
我们需要证明：**当 $u$ 被选入 $S$ 的瞬间，其当前的临时距离 $dist[u]$ 就是从 $s$ 到 $u$ 的真实最短距离。**

##### 数学归纳法证明
*   **初始状态**：$S = \{s\}$，有 $dist[s] = 0$，显然正确。
*   **归纳假设**：假设当前的集合 $S$ 中所有顶点的距离值 $dist$ 均已是真实最短距离。
*   **步骤证明**：
    假设算法在当前步中，选择了 $V-S$ 中 $dist$ 值最小的顶点 $u$，准备将其并入 $S$。
    我们要证明：不可能存在另一条从 $s$ 到 $u$ 的路径 $P'$ 满足 $W(P') < dist[u]$。
    
    使用反证法：假设存在这样一条更短的路径 $P'$，其长度 $W(P') < dist[u]$。
    由于路径 $P'$ 的起点 $s \in S$，终点 $u \in V-S$，该路径在从 $S$ 延伸到 $V-S$ 的过程中，必然存在**第一次**跨越割 $(S, V-S)$ 的边，设这条边为 $(x, y)$，其中 $x \in S, y \in V-S$。

    ```
            S          |        V-S
          ( s )        |
            \          |
             \         |
            ( x )=====>|====>( y )
                       |       \
                       |        \
                       |       ( u )  <-- 当前被选中的 dist 最小的顶点
                       |
    ```

    根据归纳假设，$x \in S$，所以从 $s$ 到 $x$ 的最短路径长度已经被正确计算并存放在 $dist[x]$ 中。
    因为我们在将 $x$ 并入 $S$ 时，必定已经对其所有出边进行了松弛操作，所以必然有：
    $$dist[y] \le dist[x] + w(x, y)$$
    沿着路径 $P'$，从 $s$ 经过 $x, y$ 到达 $u$ 的总权值为：
    $$W(P') = W(s \to x) + w(x, y) + W(y \to u)$$
    由于 $x \in S$ 且 $s \to x$ 路径全部在 $S$ 内部，根据前驱松弛结果有：
    $$W(s \to x) + w(x, y) = dist[x] + w(x, y) \ge dist[y]$$
    代入上式得到：
    $$W(P') \ge dist[y] + W(y \to u)$$
    由于**图中所有边的权值均非负**，故从 $y$ 到 $u$ 的子路径权重和必有：
    $$W(y \to u) \ge 0$$
    因此可以得到：
    $$W(P') \ge dist[y]$$
    然而，在算法步骤中，我们是在 $V-S$ 集合中选择了 $dist[u]$ 最小的顶点。这就意味着：
    $$dist[u] \le dist[y]$$
    结合上述不等式，我们得出：
    $$W(P') \ge dist[y] \ge dist[u]$$
    这与我们的反假设 $W(P') < dist[u]$ 产生了直接矛盾！
    
    因此，假设不成立。在 $V-S$ 中选择 $dist$ 最小的 $u$ 加入 $S$ 时，其 $dist[u]$ 必定已经是真实的全局最短路径。

证明完毕。

---

#### 3.1.2 为什么 Dijkstra 算法不能处理带有负权重的边？
从上述数学归纳法证明中可以看出，其最核心的一步推导是：
$$W(P') \ge dist[y] + W(y \to u) \ge dist[y]$$
这一步必须建立在 $W(y \to u) \ge 0$ 的基础之上。如果图中存在**负权边**，那么从 $y$ 到 $u$ 的子路径权重和 $W(y \to u)$ 可以是负数，上述不等式便不再成立。即使 $dist[u] \le dist[y]$，我们也完全可能通过一条经过 $y$ 后并包含极大负权边的路径到达 $u$，使得其实际路径长度小于 $dist[u]$。

##### 具体图反例跟踪模拟

为了彻底理清这一失效机制，我们构造如下含有负权边的有向图 $G$：

*   **顶点集**：$V = \{A, B, C\}$
*   **边集**：$E = \{ \langle A, B \rangle: 2, \ \langle B, C \rangle: -3, \ \langle A, C \rangle: 1 \}$
*   **源点**：$A$

```
          2
     A ------> B
     |        /
    1|      / -3
     v    v
     C <'
```

我们按照 Dijkstra 算法的执行步骤进行模拟跟踪：

1.  **初始化**：
    *   $dist[A] = 0, \ dist[B] = \infty, \ dist[C] = \infty$。
    *   已确定最短路径的集合 $S = \emptyset$。
2.  **第一轮迭代**：
    *   在未确定集合 $\{A, B, C\}$ 中选择 $dist$ 最小的节点，即顶点 $A$（$dist[A]=0$）。
    *   将 $A$ 加入已确定集合，$S = \{A\}$。
    *   松弛 $A$ 的所有出边：
        *   对边 $\langle A, B \rangle$：$dist[B] = \min(\infty, 0 + 2) = 2$。
        *   对边 $\langle A, C \rangle$：$dist[C] = \min(\infty, 0 + 1) = 1$。
3.  **第二轮迭代**：
    *   在未确定集合 $\{B, C\}$ 中选择 $dist$ 最小的节点。此时 $dist[C] = 1$ 且 $dist[B] = 2$，因此选择顶点 $C$。
    *   将 $C$ 加入已确定集合，$S = \{A, C\}$。
    *   由于 $C$ 没有出边，松弛操作结束。
4.  **第三轮迭代**：
    *   在未确定集合 $\{B\}$ 中选择最小的节点，即顶点 $B$。
    *   将 $B$ 加入已确定集合，$S = \{A, B, C\}$。
    *   松弛 $B$ 的出边：对边 $\langle B, C \rangle$ 执行松弛：
        *   如果我们尝试计算通过 $B$ 作为中继到达 $C$ 的距离：$dist[B] + w(B, C) = 2 + (-3) = -1$。
        *   此时我们发现 $-1 < dist[C]$（当前 $dist[C]=1$）。
        *   但由于 Dijkstra 的贪心策略**判定已经加入 $S$ 集合的顶点（即 $C$）的最短路径已被锁定**，因此不会再重新把 $C$ 移出 $S$ 去重新传播其最短路径变化，或者即使强行更新了 $dist[C] = -1$，如果有其他节点依赖 $C$，由于算法不再回溯更新它们，也将导致后续节点的最短距离全部出错。

最终，Dijkstra 算法输出的最短路径结果为：
*   $A \to C$ 的距离为 $1$。
*   然而，实际的最短路径应当是 $A \to B \to C$，其权重之和为 $2 + (-3) = -1$。

这说明，**负权重边直接破坏了松弛操作的贪心单调性**，使得已锁定顶点的最短路径被错误估计，从而导致整个算法失效。

---

#### 3.1.3 堆优化 Dijkstra 算法的执行步骤与复杂度优化分析

通过引入最小堆，可以将每次选择最小距离节点的时间复杂度从 $O(V)$ 降低为 $O(\log V)$，使得算法在稀疏图上拥有极高的时间效率。

##### 堆优化算法步骤：
1.  初始化 $dist$ 数组为无穷大，源点 $dist[s] = 0$。
2.  将源点对 `(0, s)` 压入最小二叉堆。
3.  只要堆不为空，重复执行：
    *   从堆顶取出距离最小的节点对 `(d, u)`，并将其从堆中弹出。
    *   **判断冗余**：如果弹出的距离 `d > dist[u]`，说明该节点在此前已经被更短的路径松弛过，当前弹出的是废弃的旧数据，直接丢弃（继续下一轮循环）。
    *   若没有冗余，说明 `dist[u]` 已经是真实最短距离，锁定 $u$。
    *   遍历 $u$ 的所有出边 $\langle u, v \rangle$，权重为 $w$：
        *   若 $dist[u] + w < dist[v]$，则执行松弛操作：更新 $dist[v] = dist[u] + w$，并将新结点对 `(dist[v], v)` 压入堆中。

##### C++ 代码实现
```cpp
#include <iostream>
#include <vector>
#include <queue>

struct Edge {
    int to;
    int w;
};

struct Node {
    int dist;
    int u;
    bool operator>(const Node& other) const {
        return dist > other.dist; // 最小堆
    }
};

std::vector<int> dijkstra(int v_count, int source, const std::vector<std::vector<Edge>>& adj) {
    std::vector<int> dist(v_count, 1e9); // 初始化为无穷大
    dist[source] = 0;

    std::priority_queue<Node, std::vector<Node>, std::greater<Node>> pq;
    pq.push({0, source});

    while (!pq.empty()) {
        Node current = pq.top();
        pq.pop();

        int u = current.u;
        int d = current.dist;

        // 核心优化：丢弃堆中的冗余过时节点
        if (d > dist[u]) continue;

        for (const auto& edge : adj[u]) {
            int v = edge.to;
            int w = edge.w;
            if (dist[u] + w < dist[v]) {
                dist[v] = dist[u] + w;
                pq.push({dist[v], v});
            }
        }
    }
    return dist;
}
```

##### 复杂度优化分析
在朴素 Dijkstra 算法中：
- 每次寻找距离最小的顶点需遍历 $V$ 个节点，共执行 $V$ 次，复杂度为 $O(V^2)$。
- 每次松弛更新 $dist$ 数组为 $O(1)$，共执行 $E$ 次，复杂度为 $O(E)$。
- 总复杂度为 $O(V^2 + E)$。对于稀疏图，由于 $E \ll V^2$，时间复杂度退化严重。

在堆优化 Dijkstra 算法中：
- 每次从堆中取出最小值需要 $O(\log(\text{堆大小}))$ 的时间。因为我们采用懒惰删除，堆中可能同时存在多个重复节点的冗余元素，堆的最大容量可达到 $O(E)$。
- 每次成功松弛一条边，就会向堆中压入一个新节点对，最多压入 $E$ 次，每次耗时 $O(\log E)$。
- 堆的弹出操作共执行 $E$ 次（包含冗余结点的弹出），每次耗时 $O(\log E)$。
- 因此，总时间复杂度为 $O(E \log E)$。由于 $E \le V^2$，$\log E \le 2\log V$，故时间复杂度可以写为：
  $$O(E \log V)$$
  对于稀疏图，堆优化 Dijkstra 的执行效率远远高于朴素版的 $O(V^2)$。

### 3.2 Bellman-Ford 算法与 SPFA 算法

当图中包含负权边时，Dijkstra 算法失效。为了解决带负权边（甚至包含负权环）的单源最短路径问题，我们需要引入 Bellman-Ford 算法及其队列优化版本——SPFA。

#### 3.2.1 Bellman-Ford 算法的核心思想与动态规划本质
Bellman-Ford 算法基于**动态规划（DP）**思想。
设 $dp[k][u]$ 表示从源点 $s$ 出发，经过**至多** $k$ 条边到达顶点 $u$ 的最短路径长度。

##### 状态转移方程：
$$dp[k][u] = \min \Big( dp[k-1][u], \min_{(v, u) \in E} \{ dp[k-1][v] + w(v, u) \} \Big)$$
其物理意义是：从源点 $s$ 到达 $u$ 且边数不超过 $k$ 的最短路径，要么是边数不超过 $k-1$ 的最短路径（不经过最新松弛的边），要么是通过某条入边 $\langle v, u \rangle$，从经过至多 $k-1$ 条边到达 $v$ 的最短路径转移而来。

由于在一个不含负权环的图中，任意两点之间的最短路径最多包含 $V-1$ 条边（若包含 $V$ 条或更多边，根据鸽巢原理，路径中必然存在至少一个环路，去掉该环路路径长度不会增加）。
因此，我们只需将外层循环的边数约束 $k$ 从 $1$ 迭代到 $V-1$，在每一轮中对图中的**所有边**进行一次松弛操作，即可求得最终的单源最短路径。由于每次迭代只与上一轮状态有关，我们可以进行空间压缩，将第一维 $k$ 省去，直接在 $dist[u]$ 数组上原位更新。

##### 负权环路检测的数学原理
如果在图 $G$ 中不存在负权环路，那么在完成了 $V-1$ 次对所有边的迭代松弛后，所有顶点均已收敛至其最终的最短路径长度。此时，对于图中的任意一条边 $\langle u, v \rangle$，都必然满足三角不等式：
$$dist[v] \le dist[u] + w(u, v)$$
如果在第 $V$ 次对所有边进行松弛遍历时，依然存在某条边满足：
$$dist[u] + w(u, v) < dist[v]$$
说明我们通过引入第 $V$ 条边（即形成了一个包含至少 $V+1$ 个顶点的路径，其必然包含一个环路），使得从源点到 $v$ 的距离变得更短了。这种“越绕越短”的环路只能是**负权环路**。

##### C++ 代码实现
```cpp
#include <iostream>
#include <vector>

struct Edge {
    int u, v, w;
};

bool bellman_ford(int v_count, int source, const std::vector<Edge>& edges, std::vector<int>& dist) {
    dist.assign(v_count, 1e9);
    dist[source] = 0;

    // 执行 V - 1 次松弛
    for (int i = 1; i <= v_count - 1; ++i) {
        bool any_update = false;
        for (const auto& edge : edges) {
            if (dist[edge.u] != 1e9 && dist[edge.u] + edge.w < dist[edge.v]) {
                dist[edge.v] = dist[edge.u] + edge.w;
                any_update = true;
            }
        }
        if (!any_update) {
            break; // 提前收敛优化
        }
    }

    // 第 V 次松弛，用于检测负权环
    for (const auto& edge : edges) {
        if (dist[edge.u] != 1e9 && dist[edge.u] + edge.w < dist[edge.v]) {
            return false; // 说明存在负权环，无法计算有效最短路
        }
    }
    return true;
}
```

##### 复杂度分析
*   **时间复杂度**：外层循环执行 $V-1$ 次，内层循环遍历所有边 $E$，因此时间复杂度是硬性的 $O(V \cdot E)$。
*   **空间复杂度**：仅需存储边集和 $dist$ 数组，空间复杂度为 $O(V + E)$。

---

#### 3.2.2 SPFA 算法（Shortest Path Faster Algorithm）
在 Bellman-Ford 算法中，每一轮迭代我们都会无条件地遍历所有边。然而在实际运行中，只有在上一轮中 $dist$ 值发生改变的节点 $u$，从它指出的边 $\langle u, v \rangle$ 才有可能在这一轮中成功松弛其邻接点 $v$。

SPFA 算法的核心思想就是：**用一个队列来维护当前可能引起松弛的活跃顶点集合**。

##### 队列优化与负环检测原理：
1.  初始将源点 $s$ 入队，并标记其“在队列中”。
2.  每次弹出队首节点 $u$，取消“在队列中”的标记。
3.  遍历 $u$ 的所有出边 $\langle u, v \rangle$，如果 $dist[u] + w(u, v) < dist[v]$：
    *   更新 $dist[v] = dist[u] + w(u, v)$。
    *   如果 $v$ 当前不在队列中，则将 $v$ 压入队列，并标记其“在队列中”。
4.  **负环检测**：为了检测负环，我们需要额外记录每个顶点入队的次数（或者记录从源点到当前顶点的最短路径所包含的边数）。如果在算法执行过程中，某个顶点的入队次数达到 $V$ 次（或最短路径边数 $\ge V$），则说明图中存在负权环路，算法应立即报错并终止。

##### C++ 代码实现
```cpp
#include <iostream>
#include <vector>
#include <queue>

struct Edge {
    int to;
    int w;
};

bool spfa(int v_count, int source, const std::vector<std::vector<Edge>>& adj, std::vector<int>& dist) {
    dist.assign(v_count, 1e9);
    dist[source] = 0;

    std::queue<int> q;
    std::vector<bool> in_queue(v_count, false);
    std::vector<int> entry_count(v_count, 0); // 记录每个顶点的入队次数

    q.push(source);
    in_queue[source] = true;
    entry_count[source] = 1;

    while (!q.empty()) {
        int u = q.front();
        q.pop();
        in_queue[u] = false;

        for (const auto& edge : adj[u]) {
            int v = edge.to;
            int w = edge.w;

            if (dist[u] + w < dist[v]) {
                dist[v] = dist[u] + w;
                if (!in_queue[v]) {
                    q.push(v);
                    in_queue[v] = true;
                    entry_count[v]++;
                    
                    // 检测负权环
                    if (entry_count[v] >= v_count) {
                        return false; // 存在负权环
                    }
                }
            }
        }
    }
    return true; // 正常结束，无负权环
}
```

##### SPFA 的算法退化与缺陷
SPFA 算法在随机稀疏图上的表现非常出色，其实际运行时间大约为 $O(k \cdot E)$（其中 $k$ 是一个与图特征相关的常数，通常 $k \le 2$）。
然而，**SPFA 的最坏时间复杂度为 $O(V \cdot E)$**。
在一些人为构造的图（如“菊花图”或“网格图”）中，通过精心排列边权，使得每次入队都会导致几乎全图节点的连锁松弛，队列会被反复塞满，算法将退化至经典的 Bellman-Ford 复杂度。

> [!WARNING]
> 在工业实践和算法选型中，若图中所有边权均为非负数，应**坚决避免使用 SPFA**，而应当首选堆优化的 Dijkstra 算法，因为后者能提供稳定的 $O(E \log V)$ 时间复杂度上限，不会被恶意构造的数据卡死。

## 4. 多源最短路径算法 (All-Pairs Shortest Path)

多源最短路径（APSP）问题的目标是求出图中任意两点 $i$ 和 $j$ 之间的最短路径。

### 4.1 Floyd-Warshall 算法

Floyd-Warshall 算法是解决多源最短路径问题的经典动态规划算法，能够处理带有负权边的图（但不能有负权环）。

#### 4.1.1 动态规划思想与状态转移方程设计
设图的顶点编号为 $1, 2, \dots, V$。
我们定义状态 $dp[k][i][j]$ 表示：**从顶点 $i$ 到顶点 $j$ 的最短路径长度，且该路径上的中间节点只能从集合 $\{1, 2, \dots, k\}$ 中选择。**

##### 状态转移推导：
当我们试图将中继节点限制从 $k-1$ 扩大到 $k$ 时，从 $i$ 到 $j$ 的最短路径有两种可能：
1.  **路径不经过节点 $k$**：此时，最短路径就是仅允许使用前 $k-1$ 个节点作为中继时的最短路径，即：
    $$dp[k][i][j] = dp[k-1][i][j]$$
2.  **路径经过节点 $k$**：因为最短路径中不包含环路，所以该路径必然可以拆分为两段：从 $i$ 到 $k$ 的路径和从 $k$ 到 $j$ 的路径。且这两段路径中的中间节点都只来自于前 $k-1$ 个节点。因此，此时的最短路径长度为：
    $$dp[k][i][j] = dp[k-1][i][k] + dp[k-1][k][j]$$

综合上述两种情况，状态转移方程为：
$$dp[k][i][j] = \min \Big( dp[k-1][i][j], \ dp[k-1][i][k] + dp[k-1][k][j] \Big)$$

##### 边界条件：
- 当 $k = 0$ 时，表示不允许任何中继节点。此时 $dp[0][i][j]$ 即为图的初始状态：若存在边 $(i, j)$，则 $dp[0][i][j] = w(i, j)$；若 $i == j$，则 $dp[0][i][j] = 0$；否则为 $\infty$。

##### 空间优化（降维）：
注意到在计算 $dp[k]$ 层状态时，只依赖于 $dp[k-1]$ 层。因此我们可以省去第一维，直接在一个二维矩阵 $dp[i][j]$ 上原位更新：
$$dp[i][j] = \min \Big( dp[i][j], \ dp[i][k] + dp[k][j] \Big)$$
为了确保这种降维是数学安全的，我们需要证明：在计算 $dp[i][j]$ 时，同一行或同一列上已被更新的值 $dp[i][k]$ 和 $dp[k][j]$，它们在第 $k$ 步与第 $k-1$ 步的值是相等的。
因为从 $i$ 到 $k$ 且以 $\{1, \dots, k\}$ 为中继的最短路径，不可能在中间阶段经过 $k$ 本身。所以：
$$dp[k][i][k] = dp[k-1][i][k]$$
这说明我们在计算 $dp[i][j]$ 时所依赖的 $dp[i][k]$ 和 $dp[k][j]$ 在本轮中是不会改变的，因而直接进行二维覆盖是完全正确的。

---

#### 4.1.2 三重循环的物理意义与顺序深究

Floyd 算法的代码实现非常简洁，主要由三重循环构成：
```cpp
for (int k = 1; k <= v_count; ++k) {
    for (int i = 1; i <= v_count; ++i) {
        for (int j = 1; j <= v_count; ++j) {
            if (dp[i][k] != 1e9 && dp[k][j] != 1e9) {
                dp[i][j] = std::min(dp[i][j], dp[i][k] + dp[k][j]);
            }
        }
    }
}
```

##### 为什么最外层循环必须是中继节点 $k$？
这是初学者最容易混淆的逻辑盲区。很多人会直觉性地写成把 $i$ 或 $j$ 放在最外层，如：
```cpp
// 错误的循环顺序！
for (int i = 1; i <= v_count; ++i) {
    for (int j = 1; j <= v_count; ++j) {
        for (int k = 1; k <= v_count; ++k) {
            dp[i][j] = min(dp[i][j], dp[i][k] + dp[k][j]);
        }
    }
}
```

##### 错误机理分析与具体反例：
动态规划的本质是**状态按阶段逐步演进**。Floyd 算法的“阶段”是**中继节点的候选集大小 $k$**。
- 当 $k$ 循环在最外层时，算法保证了在进行第 $k$ 轮计算前，所有点对之间的 $dp[i][j]$ 都已经是“仅允许使用前 $k-1$ 个节点作为中继”的最短路径。
- 如果将 $k$ 放在最内层，当我们在处理顶点对 $(i, j)$ 时，试图遍历所有可能的 $k$。但此时，我们所依赖的 $dp[i][k]$ 和 $dp[k][j]$ 还没有经过完整的以其他节点作为中继的计算，它们还停留在初始状态或不完整状态，这直接违反了动态规划的“无后效性”。

我们来看一个具体的 4 节点链状图反例：
*   **拓扑结构**：$1 \to 2 \to 3 \to 4$，各边权重均为 $1$。
*   **初始状态**：
    *   $dp[1][2]=1, \ dp[2][3]=1, \ dp[3][4]=1$
    *   其余距离均为 $\infty$。

若将 $k$ 放在最内层，执行流程如下：
1.  外层循环走到 $i = 1$。
2.  中层循环走到 $j = 4$。
3.  内层循环遍历 $k$：
    *   当 $k = 2$ 时，$dp[1][4] = \min(\infty, dp[1][2] + dp[2][4]) = \min(\infty, 1 + \infty) = \infty$。
    *   当 $k = 3$ 时，$dp[1][4] = \min(\infty, dp[1][3] + dp[3][4]) = \min(\infty, \infty + 1) = \infty$。
    由于此时 $dp[2][4]$ 和 $dp[1][3]$ 的计算还没有开始，导致 $dp[1][4]$ 无法被成功松弛，依然保持 $\infty$。
4.  整个算法结束后，我们根本无法求出 $1$ 到 $4$ 的正确最短距离（实际应为 $3$）。

因此，**必须将代表 DP 阶段的中继节点 $k$ 的循环置于最外层**。

---

#### 4.1.3 复杂度与负权环判定
*   **时间复杂度**：硬性三重嵌套循环，时间复杂度为 $\Theta(V^3)$。
*   **空间复杂度**：由于降维优化，仅需维护一个二维矩阵，空间复杂度为 $O(V^2)$。
*   **负环判定**：
    如果在 Floyd-Warshall 算法执行完毕后，检查发现存在某个对角线元素：
    $$dp[i][i] < 0$$
    这意味着从顶点 $i$ 出发，经过一系列中继节点回到顶点 $i$ 的路径总权值为负。这说明**图中存在一个包含顶点 $i$ 的负权环路**。

## 5. 算法对比与工程选型指南

在面临实际的工程研发时，我们需要综合考量多种外部约束，以选出效率最优、实现成本合理的算法。

### 5.1 核心算法多维选型矩阵

下表总结了本文所涉及的所有最短路径算法的各项关键特征：

| 算法名称 | 适用范围 | 边权约束 | 最坏时间复杂度 | 空间复杂度 | 算法本质 | 优缺点与适用场景 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| **Dijkstra (朴素版)** | 单源最短路 | 必须非负 | $O(V^2)$ | $O(V^2)$ 或 $O(V+E)$ | 贪心选择 | 适合点数少而边数极多的超稠密图。实现简单。 |
| **Dijkstra (二叉堆优化)** | 单源最短路 | 必须非负 | $O(E \log V)$ | $O(V + E)$ | 优先队列贪心 | 最常用的通用单源算法，在稀疏图与中等稠密图上表现极佳且性能稳定。 |
| **Bellman-Ford** | 单源最短路 | 无限制 | $O(V \cdot E)$ | $O(V + E)$ | 动态规划 | 效率较低，但能检测负权环。现少用，多作为 SPFA 的备用保障。 |
| **SPFA** | 单源最短路 | 无限制 | $O(V \cdot E)$，随机图 $O(kE)$ | $O(V + E)$ | 队列优化松弛 | 带负权图的首选。但需提防被特殊数据卡到 $O(VE)$ 最坏复杂度。 |
| **Floyd-Warshall** | 多源最短路 | 允许负权，不能有负环 | $O(V^3)$ | $O(V^2)$ | 动态规划 | 代码实现极其简单。适合小规模点数（$V \le 500$）的多源路径计算。 |

### 5.2 复杂工程场景下的变体与优化思考

在解决更复杂的真实工业系统问题时，往往需要在基础算法之上进行优化与变体设计。

#### 5.2.1 双向 Dijkstra 算法 (Bidirectional Dijkstra)
对于超大规模的路网规划，普通的 Dijkstra 算法会像波纹一样向四周辐射扩散，搜索空间呈 $O(\pi r^2)$ 扩张。
**双向 Dijkstra** 从源点 $s$ 和终点 $t$ 同时启动搜索：
- 源点 $s$ 进行正向 Dijkstra，终点 $t$ 进行反向（沿边反向）Dijkstra。
- 两个方向轮流进行松弛。当某一个顶点在正向和反向搜索中均被标为“已确定最短路径”时，算法终止。
- 此时，搜索空间被压缩为两个半径为 $r/2$ 的圆，总搜索面积降为：
  $$\pi (r/2)^2 + \pi (r/2)^2 = \frac{1}{2} \pi r^2$$
  理论上减少了一半的搜索节点，效率提升非常明显。

#### 5.2.2 启发式搜索 (A* 算法)
当我们需要快速定位方向明确的路径（如地图导航）时，可以引入启发式评估函数 $h(n)$。
A* 算法每次选择 $f(n) = g(n) + h(n)$ 最小的节点进行扩展。其中 $g(n)$ 是源点到当前节点的实际代价，而 $h(n)$ 是当前节点到终点的估算代价（如欧几里得距离或曼哈顿距离）。
- A* 可以看作是带有“势能函数”重加权的 Dijkstra 算法。
- 在满足一致性（Consistency）的前提下，A* 算法能极大地朝着终点方向收窄搜索扇面，是游戏寻路与自动驾驶规划的核心算法。

#### 5.2.3 稀疏图多源最短路径选型：Dijkstra vs Floyd
若我们需要求一个稀疏图（如 $E = O(V)$）的任意两点间最短路径：
- 若运行 Floyd 算法，时间复杂度是硬性的 $O(V^3)$。
- 若我们对图中的每个顶点，都分别运行一次堆优化 Dijkstra 算法：
  由于单次 Dijkstra 复杂度为 $O(E \log V)$，执行 $V$ 次的总复杂度为：
  $$O(V \cdot E \log V)$$
  在稀疏图下，该复杂度等价于 $O(V^2 \log V)$，当 $V = 2000$ 时：
  - $V^3 = 8 \times 10^9$（普通 CPU 运行需数秒到数十秒）。
  - $V^2 \log V \approx 4 \times 10^6 \times 11 \approx 4.4 \times 10^7$（仅需几十毫秒）。
  - 两者性能差距达到数个数量级。

#### 5.2.4 负权多源最短路：Johnson 算法
如果多源最短路计算中包含负权边，不能直接对每个顶点跑 Dijkstra 算法。但如果使用 Floyd，复杂度又过高。
**Johnson 算法** 巧妙地结合了 Bellman-Ford 与 Dijkstra：
1.  建立一个虚拟源点 $s'$，向图中所有顶点连一条权重为 $0$ 的边。
2.  用 Bellman-Ford 算法计算 $s'$ 到所有顶点的最短路径 $h(u)$。如果检测到负环，算法退出。
3.  利用计算出的 $h(u)$ 重新给图中的每条边 $\langle u, v \rangle$ 重构非负权重：
    $$\hat{w}(u, v) = w(u, v) + h(u) - h(v) \ge 0$$
4.  基于非负的新权重 $\hat{w}$，对每个顶点运行一次堆优化的 Dijkstra 算法。
5.  最后将计算出的距离通过公式逆推回原图的真实距离：
    $$dist(u, v) = \widehat{dist}(u, v) + h(v) - h(u)$$
- Johnson 算法在带负权稀疏图上的总时间复杂度为 $O(VE + V^2 \log V)$，完美填补了 Floyd 算法在稀疏带负权图上的效率缺陷。

#### 5.2.5 现代网络路由协议中的应用
在计算机网络中，路由协议的底层核心就是图最短路径算法：
- **链路状态路由协议（如 OSPF、IS-IS）**：每个路由器在本地维护整个网络拓扑的完整“邻接表”（即链路状态数据库 LSDB），然后以自身为源点，运行 **Dijkstra 算法** 计算到达所有网段的最短路径树，更新路由表。
- **距离矢量路由协议（如 RIP）**：路由器只知道相邻路由器发来的距离信息，通过定期交换距离矢量，以**分布式 Bellman-Ford 算法**不断更新自己到达目的网络的距离。这种分布式松弛策略极易产生“路由环路”与“计数到无穷”问题，需要通过水平分割、毒性逆转等技术予以修正。

综上所述，经典图算法及其底层的数学推导与演进不仅活跃在理论计算机科学中，更是现代分布式系统、网络工程与空间数据处理不可或缺的技术底座。在工程实践中，只有深刻把握算法的数学证明与计算机表示结构，才能在复杂多变的场景中游刃有余地进行算法选型与系统调优。
